# 店長 AI POS 產品藍圖

## 1. 產品定位

店長 AI POS 是一套低硬體門檻、低月租、可快速上線、功能深度接近高價 POS，並內建 AI 營運輔助的店務作業系統。

對外一句話：

> 一套系統，搞定收銀、點餐、發票、庫存、會員、行銷、報表與老闆決策，月租從小店負擔得起的價格開始。

真正差異化不是「功能也有」，而是：

- 上線快：拍菜單、AI 建檔、測試出單、開店。
- 費用低：小店能先用，不被高月租嚇跑。
- 深度夠：成長後不必從低價接單工具換到另一套 POS。
- 現場穩：斷網可營業、出單穩、發票安全、同步可追蹤。
- 決策有用：不是給一堆報表，而是告訴老闆明天怎麼賣。

## 2. 市場切入

台灣餐飲與零售店家已普遍需要 POS、電子發票、線上接單與會員工具。市場兩端都已有人：

- 完整型 POS：功能成熟，但常落在每月約 NT$1,500 到 NT$3,000 以上，且進階模組、設備或導入服務會增加總成本。
- 低價接單工具：月租可低到 NT$399 左右，但常以接單、掃碼點餐、LINE 點餐為主，完整收銀、電子發票、庫存、權限與對帳深度可能不足。

切入策略：

> 用低價打進小店，但底層架構不能做成玩具。Starter 可以功能少，架構必須能長成 Pro 和 Chain。

## 3. 第一波目標客戶

### 早餐店、飲料店、小吃店、便當店

痛點：尖峰排隊、電話接單錯誤、客製選項多、人手少、害怕高月租、不想被外送平台抽成。

主打：

- QR / LINE 點餐
- 自動出單
- 飲料甜度冰量、早餐套餐、便當配菜模板
- 低月租、不抽成、不綁硬體
- LINE 老闆日報

### 咖啡廳、早午餐、正餐小餐廳

痛點：桌位、分帳、內用外帶外送混亂、廚房漏單、交班對帳、毛利不透明。

主打：

- POS 收銀
- 桌位與 KDS
- 會員與優惠券
- 庫存與日報
- 電子發票健康檢查

### 小型零售店、選物店、服飾店、寵物用品店

痛點：SKU 多、庫存不準、退換貨複雜、會員資料分散、線上線下庫存不同步。

主打：

- 條碼收銀
- 庫存與進銷存
- 手機盤點
- 會員與優惠券
- 商品資料匯入

### 成長型連鎖與加盟總部

痛點：分店價格不同步、加盟主亂改商品、活動不能統一派送、總部看不到即時營收、多店庫存與調撥混亂。

主打：

- 總部控制台
- 分店權限
- 中央菜單與價格控管
- 跨店報表
- 活動派送
- 加盟稽核

## 4. 核心產品模組

### A. POS 前台收銀

P0 必備：

- PIN / 帳密登入
- 開班與交班
- 商品分類、搜尋、快速鍵
- 購物車、數量、備註、客製選項
- 單品折扣、整單折扣、主管折扣
- 現金、刷卡記錄、行動支付記錄、混合付款
- 發票資訊：統編、載具、捐贈碼
- 收據與廚房單列印
- 退貨、作廢、補印與稽核紀錄
- 基本離線交易
- 訓練模式

前台 UI 原則：

- 左側分類、中間商品、右側購物車、底部付款。
- 尖峰流程優先，不做 ERP 式複雜畫面。
- 新人 10 分鐘可上手。

### B. 線上點餐與掃碼點餐

必備：

- QR Code 內用點餐
- 外帶預訂
- LINE 點餐入口
- 客製選項：冰量、甜度、加料、辣度、備註
- 線上付款狀態記錄
- 叫號與 LINE 通知
- 缺貨下架
- 黑名單與棄單控管

亮點：

> 不用買大型 kiosk，一台平板就能變自助點餐機。

### C. 訂單中台 Order Hub

所有訂單來源進同一個池：

- POS 現場單
- QR Code 單
- LINE 單
- 電話手動建單
- 外送平台單
- 自助機單
- 電商單
- 團購單
- 預約單

每張單必須追蹤：

- 訂單來源
- 付款狀態
- 製作狀態
- 預計完成時間
- 出單工作站
- 發票狀態
- 庫存扣減狀態

### D. 電子發票模組

電子發票要做成信任功能，不只是「可以開」。

必備：

- 字軌匯入與分配
- 字軌快用完提醒
- B2C / B2B 發票
- 手機條碼、自然人憑證、會員載具、捐贈碼
- 統編檢查
- 作廢與折讓流程
- 上傳佇列
- 漏傳、重號、失敗警示
- 日結比對：訂單金額、付款金額、發票金額
- MIG / Turnkey 版本追蹤

對外說法：

> 別人只是幫你開發票，我們幫你避免發票出事。

### E. 付款與對帳

支付策略：POS 不碰完整卡號與敏感卡片資料。刷卡與行動支付由合格金流、刷卡機或第三方 SDK 處理，POS 保存交易序號、授權碼、金額、狀態與對帳資料。

對帳要能比對：

- POS 應收
- 現金實收
- 刷卡授權
- 金流請款
- 退款紀錄
- 發票金額
- 銀行入帳

核心亮點：

```text
銷售訂單金額 = 付款成功金額 = 發票開立金額 = 金流入帳金額
```

不一致就提醒。

### F. 庫存與進銷存

P1：

- 即時扣庫存
- 退貨回補
- 進貨
- 盤點
- 安全庫存
- 報廢
- 平均成本

P2：

- 門市調撥
- 批號與效期
- BOM 食材配方
- 理論庫存與實際庫存差異
- 智慧備料

智慧備料早期應先用規則與歷史均值，不要一開始承諾天氣、節日、活動全自動模型。

### G. 會員 CRM 與行銷

必備：

- 手機會員
- LINE 綁定
- 消費紀錄
- 點數
- 優惠券
- 生日禮
- 會員載具
- 黑名單

進階：

- 會員等級
- 儲值金
- 回購券
- 尖峰分流促銷
- 庫存出清促銷
- AI 會員召回建議

### H. 後台管理

核心後台：

- 門市管理
- 員工管理
- 角色與權限
- 商品、SKU、分類、圖片
- 價格表
- 菜單與客製選項
- 促銷與優惠券
- 訂單查詢
- 發票管理
- 報表中心
- 金流對帳
- 稽核紀錄
- 系統設定

連鎖後台：

- 中央菜單
- 中央價格
- 分店覆蓋規則
- 活動派送
- 跨店報表
- 加盟稽核
- 分店權限限制

### I. AI 營運助理

AI 不應只做聊天框，而要做每天有用的營運輸出。

第一階段：規則型 + LLM 摘要

- 今日營收摘要
- 熱銷商品
- 付款方式分布
- 作廢與退貨異常
- 庫存不足
- 發票上傳異常
- 明日待辦清單

第二階段：分析建議

- 低毛利高銷量商品提醒
- 促銷建議
- 會員召回建議
- 尖峰人力建議
- 商品漲價候選

第三階段：預測模型

- 備料預測
- 等待時間預估
- 匿名同業比較
- 展店與區域分析

## 5. 收費策略

價格原則：核心低價，進階加值，服務分級，硬體不綁，導入自動化。

建議方案：

| 方案 | 月費 | 對象 | 核心功能 |
| --- | ---: | --- | --- |
| Free 試用版 | NT$0 | 測試店家 | 商品限制、交易限制、無正式電子發票 |
| Starter 小店版 | NT$399-599 | 早餐、飲料、小吃 | POS、掃碼點餐、LINE 接單、基本報表 |
| Basic 標準版 | NT$799-999 | 一般店家 | 電子發票、會員、優惠券、基本庫存 |
| Pro 專業版 | NT$1,299-1,599 | 正餐、零售、成長店 | 桌位、KDS、進銷存、AI 日報、對帳 |
| Chain 連鎖版 | NT$2,999 起 | 多店、加盟 | 總部管理、多店報表、權限、API |
| Enterprise | 客製 | 大型品牌 | SLA、私有部署、ERP 串接、專屬導入 |

重要限制：

- Starter 不應承諾高成本客服。
- 電子發票若使用加值中心或需代維運，應獨立估算成本。
- AI 進階功能應以加購或 Pro 以上承載。
- 多店、API、進階庫存、導入服務是主要毛利來源。

## 6. MVP 範圍

MVP 目標：讓小店能在 7 天內正式營業使用。

P0 必做：

- POS 商品、購物車、結帳
- 現金與手動刷卡記錄
- QR / LINE 點餐
- 收據與廚房單
- 商品、分類、菜單後台
- 日營收與付款方式報表
- 開班、交班、現金盤點
- 基本離線交易
- 手機會員與消費紀錄
- 遠端診斷與錯誤回報
- 電子發票串接驗證與上線流程設計

P0 不建議做：

- 完整 BOM 備料預測
- 匿名同業比較
- 多品牌雲端廚房
- 複雜加盟合規
- 外送平台全自動深度串接
- 企業 API Marketplace
