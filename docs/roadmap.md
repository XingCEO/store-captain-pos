# 產品路線圖

## 0-1 個月：定義與驗證

目標：證明需求是真的，並把 MVP 範圍縮到可交付。

> 進度：🟡 Partial — 文件面齊全；市場/硬體驗證待補

工作：

- 訪談 30 家店：早餐、飲料、小吃、便當、咖啡。
- 建立競品表：iCHEF、SWD Order、肚肚、快一點、Eats365、Simpos、new2POS、MENU爸、OKSHOP 等。
- 定義第一個店型模板：飲料店或早餐店二選一。
- 畫 POS 前台 wireframe。
- 畫點餐流程與出單流程。
- 確認電子發票合作路線：加值中心優先或自建 Turnkey。
- 確認金流合作限制與 PCI 邊界。
- 測試 3-5 款印表機與錢櫃。

交付物：

- MVP PRD
- 系統架構圖
- 資料模型 v0
- 發票與付款風險文件
- 第一版價格頁草稿

## 2-3 個月：MVP 開發

目標：讓試點店可以完成現場收銀與線上接單。

> 進度：🟡 Partial — demo 級 API + 前端齊全；DB/正式部署待補

P0 功能：

- POS 前台：商品、購物車、結帳、現金付款、手動刷卡記錄。
- 商品後台：分類、商品、價格、客製選項。
- QR 點餐：菜單、購物車、送單。
- LINE 入口：基本導流或 LIFF PoC。
- 出單：收據、廚房單。
- 開班交班：備用金、現金盤點、付款方式對帳。
- 報表：日營收、付款方式、商品排行。
- 同步：本機 outbox、雲端 idempotency。
- 遠端診斷：版本、同步狀態、印表機狀態。

不做：

- 完整電子發票正式上線。
- 複雜庫存。
- AI 備料。
- 多店總部。

## 4-5 個月：台灣營業必備

目標：從「可試用」變成「可正式營業」。

> 進度：🟡 Partial — sandbox stub 在；正式發票/上線強化未啟動

功能：

- 電子發票正式串接或加值中心整合。
- 發票健康儀表板。
- 基本會員：手機、LINE、消費紀錄。
- 優惠券：滿額折、序號券。
- 基本庫存：銷售扣庫存、低庫存提醒。
- 離線模式強化：待同步提示、重試、dead letter。
- 權限與稽核：作廢、退款、折扣、開錢櫃。
- LINE 老闆日報：營收、異常、庫存、發票。

試點標準：

- 10 家店連續營業 30 天。
- 無遺失訂單。
- 無重複付款。
- 發票上傳異常可追蹤與修復。
- 尖峰流程可用。

## 6-8 個月：形成競爭力

目標：從工具變成可銷售產品。

> 進度：⛔ Not started — go-gate 未開

功能：

- KDS 廚房螢幕。
- 平板 kiosk 自助點餐。
- AI 日報 v1：規則型洞察 + LLM 摘要。
- 進階報表：毛利、退貨作廢、員工操作。
- 進銷存 v1：進貨、盤點、報廢。
- 多店 v1：多門市報表、中央商品。
- 一鍵搬家：Excel / CSV 商品與會員匯入。
- 行業模板：早餐、飲料、便當、咖啡、零售。

商業目標：

- 50-100 家付費店。
- Starter 到 Basic / Pro 升級路徑成立。
- 客服問題可分類與自助化。

## 9-12 個月：擴張版本

目標：建立差異化與高 ARPU 功能。

> 進度：⛔ Not started

功能：

- AI 備料 v1。
- AI 促銷建議。
- AI 會員召回。
- 總部加盟系統。
- 跨店庫存與調撥。
- API v1：會計、ERP、電商。
- 硬體健康監控。
- 匿名同業比較 PoC。

商業目標：

- 連鎖與加盟客戶開始導入。
- Pro / Chain 成為主要毛利來源。
- 建立認證硬體與導入夥伴網路。

## 每階段驗收指標

### 現場速度

- 一般商品加入：3 秒內完成。
- 結帳流程：尖峰可連續操作不卡頓。
- 飲料客製選項：不超過 2 層選單。

### 可靠性

- 訂單不得遺失。
- 同步重試不得重複建單。
- 付款狀態不得與訂單狀態不一致。
- 發票異常必須可追蹤。

### 客服成本

- 80% 常見問題可由文件或 LINE bot 回答。
- 遠端診斷可在 1 分鐘內看到店家同步、硬體、版本、發票狀態。

### 商業

- 免費試用到付費轉換率。
- Starter 到 Basic / Pro 升級率。
- 平均每店客服時數。
- 每店毛利。
- 30 / 90 天留存。
